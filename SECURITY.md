# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest (main branch) | Yes |

## Reporting a Vulnerability

This is a documentation repository and does not contain executable software. However, if you discover a security concern — such as exposed credentials, malicious links, or unsafe code examples in the guides — please report it responsibly.

### How to Report

1. **GitHub Security Advisories** (preferred): Use the [Security Advisories](https://github.com/kanjseok/claude-code-remotion-guide/security/advisories) tab to privately report the issue.
2. **GitHub Issues**: For non-sensitive concerns (e.g., unsafe code patterns in examples), open a [regular issue](https://github.com/kanjseok/claude-code-remotion-guide/issues).

### Response Timeline

- **Acknowledgment**: Within 48 hours of report submission
- **Assessment**: Within 7 days
- **Resolution**: Within 30 days for confirmed issues

## Disclosure Policy

- Report privately first using the methods above.
- Allow reasonable time for maintainers to assess and address the issue before public disclosure.
- Credit will be given to reporters in the fix commit or release notes, unless anonymity is requested.

## Security Update Process

1. Maintainer confirms the issue and assesses severity.
2. A fix is committed to the `main` branch.
3. A summary is posted in the commit message and, if warranted, in the repository's release notes.
